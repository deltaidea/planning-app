Задание

Реализовать приложение для планирования встреч с клиентами.

Условия

- чистый, читабельный, production ready код ;)
- pixel perfect верстка;
- React + Redux;
- только стандартный ES2015, никаких ES2016+ (babel можно не использовать);
- для календаря библиотеки не использовать, но можно использовать moment.js;
- для autocomplete библиотеки не использовать;
- для сборки проекта можно использовать любой bundler;
- собранный сайт должен отдаваться node.js сервером (можно использовать express и тп);
- для стилей можно использовать чистый css или любой пост процессор (less/sass/stylus);
- все данные должны храниться в local storage;
- все hover/active состояния у элементов управления можно сделать по своему усмотрению;
- данные для страницы My Clients нужно захардкодить и хранить в local storage;

Примечания к дизайну

- для формы создания/редактирования новой встречи должна присутствовать валидация isRequired для полей Participant, Description;
- числа в календаре кликабельны (при клике показывается список встреч на этот день в правой части);
- панель навигации и панель с календарем имеют фиксированную ширину, панель со списком встреч и редактором имеют динамическую ширину;
- в календаре желтая дата это текущая дата, серая дата это выделенная дата;
- зеленые кружечки под числами в календаре это индикатор наличия встреч в эти дни (если создать - встречу на этот день, соотвественно кружочек должен появится; если удалить все встречи, индикатор должен пропасть);
- обратите внимание на наличие индикатора общего количества встреч на панели навигации;
- если открыт редактор встречи и произошел переход в календаре на другую дату -> редактор закрываем, показываем выбранный день;
